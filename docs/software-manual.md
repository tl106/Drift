# Drift睡眠守护软件 V1.0 软件说明书

## 一、软件概述

### 1.1 软件名称
Drift睡眠守护软件 V1.0

### 1.2 软件用途
Drift是一款Android平台的睡眠辅助工具应用。用户在睡前收听视频、音乐、播客等媒体内容时，Drift在后台监测用户的活动状态。当检测到用户长时间无活动（即可能已入睡）后，自动渐弱系统媒体音量并暂停正在播放的媒体应用，避免手机播放一整夜导致电量耗尽。

### 1.3 开发环境
- 编程语言：Kotlin
- UI框架：Jetpack Compose + Material3
- 数据存储：Room数据库 + SharedPreferences
- 最低支持：Android 8.0（API 26）
- 目标版本：Android 15（API 35）

### 1.4 运行环境
- 操作系统：Android 8.0及以上
- 设备要求：支持加速度传感器的Android手机
- 存储空间：约2MB
- 网络要求：无需联网（纯本地运算）

## 二、功能说明

### 2.1 核心功能：智能入睡检测与媒体暂停

**功能描述：** 用户点击"开始"按钮后，应用进入后台持续监听多种活动信号。当所有信号连续沉默超过用户设定的等待时间（默认25分钟），系统判定用户可能已进入深度睡眠，随后在设定的渐弱时间内（默认10分钟）线性降低系统媒体音量至零，最后通过MediaSession API暂停正在播放的外部媒体应用。

**监听的活动信号：**
- 加速度传感器：检测手机运动（翻身、拿起手机等）
- 屏幕状态：检测屏幕亮起事件
- 系统音量：检测用户手动调节音量
- 摇一摇手势：检测用户摇晃手机

**操作流程：**
1. 打开Drift应用
2. 点击主界面中央的"开始"按钮
3. 切换到任意媒体应用（如B站、YouTube、网易云音乐等）播放内容
4. 正常入睡，Drift在后台自动完成检测与暂停

### 2.2 活动中断恢复

**功能描述：** 在音量渐弱过程中，如果检测到用户仍有活动（翻身、触屏、调音量等），系统立即停止渐弱，并在10秒内渐进恢复音量至原始值，然后重新开始等待计时。

### 2.3 摇一摇恢复

**功能描述：** 用户在任何时候摇晃手机，即可恢复音量并重新开始计时。系统通过振动反馈告知用户操作已生效。此功能允许用户在黑暗中无需亮屏即可操作。

### 2.4 自定义参数设置

**功能描述：** 用户可在设置页面调整两个核心参数：
- 等待时间：15 / 20 / 25 / 30 / 40 分钟可选
- 渐弱时间：5 / 10 / 15 / 20 分钟可选

设置即时生效，持久保存。

[截图：设置页面]

### 2.5 每晚自动启动

**功能描述：** 用户可设定每天自动启动检测的时间（21:30 / 22:00 / 22:30 / 23:00 / 23:30可选）。开启后，每晚到达设定时间时，应用自动启动检测服务，无需用户手动操作。手机重启后自动恢复定时任务。

### 2.6 睡眠记录

**功能描述：** 每次成功检测并暂停媒体后，系统自动记录：
- 检测开始时间
- 媒体暂停时间
- 被暂停的应用名称

记录按日期分组展示，存储在设备本地Room数据库中。

[截图：记录页面]

### 2.7 睡眠报告分享

**功能描述：** 用户可将任意一条睡眠记录生成为精美的图片卡片，通过系统分享功能发送到微信、朋友圈、微博等社交平台。卡片包含检测时间、守护时长、暂停的应用名称等信息。

### 2.8 桌面小组件（Widget）

**功能描述：** 提供2×1尺寸的桌面小组件，用户可在手机桌面一键启动或停止检测，无需打开应用。小组件实时显示当前运行状态。

### 2.9 厂商省电适配

**功能描述：** 针对华为、小米、OPPO、vivo、三星等主流Android厂商的后台管理策略，在设置页面提供对应的操作引导，帮助用户关闭电池优化以确保后台服务正常运行。

[截图：设置页面 - 后台运行引导]

### 2.10 通知管理

**功能描述：**
- 运行中：显示常驻通知，实时展示监听状态和已安静时间，提供"停止"快捷按钮
- 检测完成后：推送睡眠报告通知，告知用户暂停的时间和应用名称

## 三、界面说明

### 3.1 首页
首页采用深色主题设计，中央为圆形启动/停止按钮。运行时按钮变为红色并显示呼吸脉动光晕动画，营造安静的睡眠氛围。底部为提示文字。

### 3.2 记录页
以列表形式按日期分组展示历史检测记录。每条记录显示开始时间、暂停时间和被暂停的应用名称。右侧提供分享按钮。

### 3.3 设置页
分为四个区域：等待时间设置、渐弱时间设置、自动启动设置、后台运行设置。采用芯片选择器（Chip）交互方式。

### 3.4 底部导航
三个标签：首页（月亮图标）、记录（时钟图标）、设置（齿轮图标）。选中态为琥珀色，未选中为灰色。

## 四、技术架构

### 4.1 前台服务（DriftService）
使用Android前台服务（Foreground Service）实现后台持续运行，服务类型为mediaPlayback。通过Handler定时检查活动状态，控制音量渐弱和媒体暂停。

### 4.2 媒体控制（MediaControlService）
继承NotificationListenerService，通过MediaSessionManager获取当前活跃的媒体会话，调用TransportControls.pause()暂停播放。

### 4.3 数据持久化
- Room数据库：存储睡眠检测记录（SleepRecord实体）
- SharedPreferences：存储用户设置（等待时间、渐弱时间、自动启动配置）

### 4.4 传感器管理
通过SensorManager注册加速度传感器监听，使用SENSOR_DELAY_NORMAL采样率。结合BroadcastReceiver监听屏幕状态和音量变化。

## 五、隐私与安全

- 所有数据处理在设备本地完成
- 不收集、不上传任何个人信息
- 不包含第三方SDK或广告SDK
- 通知访问权限仅用于媒体暂停控制，不读取通知内容
- 加速度传感器数据仅在内存中实时处理，不存储
